Traditional Assistant vs Claude Code (Agent)

Dimension Traditional Chat Assistant Claude Code (Agent)
Work Mode Answers questions, then waits Works autonomously, executes continuously
Capability Text replies only Takes actions: edit, search, run, deploy
Context Sees only the current single file Cross-project, cross-file global understanding
Usage You write code, AI reviews it You describe the goal, AI plans and implements
Feedback Loop Relies on user judgment Self-verifying: runs tests, compares output

Agentic Loop (ReAct Paradigm)

┌─────────────────────────┐
│ 1. Gather Context       │
│ Search files, understand structure │
│          ↓            │
│ 2. Take Action         │
│ Edit files, run commands        │
│          ↓            │
│ 3. Verify Results       │
│ Run tests, confirm correctness   │
│          ↓            │
│    ↩ loop back        │
└─────────────────────────┘

KEY INSIGHT

Non-linear — iterative self-correction

Each tool call result

feeds back into the loop,
guiding the next decision

User always stays in the loop

Can interrupt and redirect at any time

Context Management — Three-Level Strategy

LEVEL 1  Fundamentals
/clear Full reset when switching tasks /compact Manual compression, can specify focus to retain
LEVEL 2  Intermediate
CLAUDE.md <200 lines Keep project instructions concise, avoid overload Auto Memory Persist notes across sessions /btw Side question, not written to history
LEVEL 3  Advanced
Subagent delegation Offload heavy tasks to isolated context Skills lazy-load Load on demand, no permanent footprint Pipeline state JSON state persists phase information Multi-session Split long tasks across sessions
