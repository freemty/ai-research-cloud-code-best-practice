Hook — Event-Driven Automation

Shell scripts triggered by specific events — deterministic execution, not suggestions

Type When Triggered Capabilities
PreToolUse Before tool execution Block + validate + modify parameters
PostToolUse After execution Format + check + advance pipeline
Stop About to stop Prevent stop + inject reminders
PreCompact Before compaction Set markers + save state

Source: code.claude.com/docs/en/hooks-guide

Hook in Practice: Experiment Pipeline

dev ──→ skill-update ──→ experiment ──→ monitoring ──→ analysis ──→ commit ──→ dev (cycle) █
  • Stop hook detects 10+ file edits → reminds to run /writing-skills
  • PostToolUse detects Skill completion → automatically advances to next stage
  • Stop hook detects experiment complete → reminds /slides-dispatch
  • Manual control: pipeline-ctl.sh {status|set|skip|reset}

Hook in Practice: Safety Guardrails

  • slides-guard.sh — PreToolUse
    Intercepts direct calls to heavy skills, recommends using dispatcher
    [slides-guard] Direct use of /agent-slides would fill main context.
    Use /slides-dispatch instead — delegates to subagent automatically.
  • console.log audit — Stop hook
    Audits all modified files before session ends, blocks commits with leftover console.log
  • git push review — PreToolUse
    Opens diff in editor for human review before push, executes only after confirmation

Sub-agent — Specialist with Independent Context

  • Independent context window + system prompt + tools
    Main context stays clean; heavy tasks complete in a disposable context
  • Built-in types:
    Explore (Haiku, fast) Plan General-purpose Custom
  • ⚠ Cannot spawn other subagents — prevents infinite nesting and resource explosion

Source: code.claude.com/docs/en/sub-agents

Before / After: Paper Reproduction

IDE Assistant
  1. Read paper
  2. Ask AI to explain
  3. Manually paste code
  4. Run → errors
  5. Ask AI to fix
  6. Repeat steps 4–5
  7. Manually start training
  8. Manually monitor progress
8 steps, manual intervention throughout, context frequently lost
Claude Code Agent
$ Please reproduce the experiments from this paper
  • Autonomously explores codebase + reads paper
  • Autonomously implements model architecture
  • Autonomously runs tests + debugs
  • Autonomously starts training + Loop monitoring
  • Autonomously generates slides + changelog
1 prompt → fully autonomous execution

Before / After: Experiment Lifecycle

Traditional Workflow
  • Manually tracking experiment state
    Spreadsheets/notes, easy to miss things
  • Context lost after session ends
    Must re-explain background next time
  • No automatic reminder for next step
    Memory-dependent, critical steps easily forgotten
  • Results scattered everywhere
    slides / logs / notes in different places
Pipeline + Hooks Workflow
  • Pipeline hooks auto-track stages
    state.json persisted, queryable anytime
  • Memory retains knowledge across sessions
    CLAUDE.md + skills never forgotten
  • Stop hook auto-reminds next step
    Deterministic execution, no memory dependency
  • Auto-generates slides + changelog
    Output ready when experiment ends, unified archive
