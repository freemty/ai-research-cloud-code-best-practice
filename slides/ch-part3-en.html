CLAUDE.md Best Practices

DO include
  • Build commands (build / test / lint)
  • Code style preferences (const/let, types)
  • Testing guidelines (framework, coverage requirements)
  • Architectural decisions (directory structure, layering rules)
DON'T include
  • Things Claude can infer on its own
  • Standard industry conventions (universally known)
  • Detailed API docs (use Skills instead)
  • Frequently changing configs (go stale quickly)
Hooks > CLAUDE.md (100% enforced, cannot be bypassed)   |   Skills > CLAUDE.md (occasional knowledge, loaded on demand)

Source: code.claude.com/docs/en/best-practices

9 Coding Agent Principles

Less is More
Skip the plugins — basic CLI is enough
Context is Everything
Separate research and implementation phases, precise instructions
Use Sycophancy
Use neutral prompts to prevent agent hallucination
Avoid Fill-in-the-Blank
Set rules in CLAUDE.md to prevent wild guessing
Define the Finish Line
Test-driven / CONTRACT.md acceptance criteria
CLAUDE.md as Logic Index
Write IF-ELSE tree pointing to rule files, not an encyclopedia
Rules vs Skills
Rules = preferences; Skills = execution recipes
Regular Cleanup
Let the agent self-organize, removing conflicting rules
Don't Chase New Tools
Official integrations are what actually matter

Boris Cherny 10 Tips (1–5)

  • 1. git worktree for parallel sessions — Run multiple Claude instances simultaneously without interference
  • 2. Plan Mode before implementing — One Claude writes the plan, another Claude audits it
  • 3. Update CLAUDE.md after every correction — Write the rule in, prevent the same class of error from recurring
  • 4. Turn daily operations into Skills — Any operation used >1x per day becomes a custom slash command
  • 5. Slack MCP lets Claude fix bugs independently — Paste bug thread, let Claude autonomously inspect CI logs

Source: @bcherny (Boris Cherny) Twitter thread, 2026-01-31

Boris Cherny 10 Tips (6–10)

  • 6. Claude as reviewer to challenge code — Have Claude play the reviewer, demand a more elegant rewrite
  • 7. Ghostty terminal + voice input — Better rendering + voice input makes prompts more detailed
  • 8. "use subagents" to invest more compute — Append the keyword, keep main context clean
  • 9. Use BigQuery and other CLIs directly for analysis — No hand-written SQL needed; Claude operates databases for real-time analysis
  • 10. Generate HTML presentations to learn code — Have Claude generate visual explanations, ASCII architecture diagrams for comprehension

Common Failure Patterns

Failure Pattern Problem Solution
Kitchen Sink Session Mixing unrelated tasks into the same session, polluting context Run /clear before switching tasks
Repeated Corrections Failed attempts pile up, context filled with error history After 2 corrections, run /clear and give a fresh context
CLAUDE.md Too Long Too much content gets ignored, conflicting rules can't be enforced Ruthlessly trim, switch to IF-ELSE logic index + Skills
Trust-Verification Gap Edge cases not covered, agent quietly goes off the rails Always provide verifiable completion criteria (CONTRACT.md)
Infinite Exploration Still exploring aimlessly as context nears exhaustion Narrow task scope, or isolate exploration with a subagent
