01

Same Model, Different Harness

Cursor, Codex, Claude Code 使用相同的底层模型，但 harness 设计决定了能力边界。

IDE-Embedded Assistant

  • — Single-turn Q&A, user-driven
  • — Text generation only
  • — Scoped to current file / selection

CLI Agent (ReAct Loop)

  • — Autonomous loop: plan → act → verify
  • — Tool calls: edit, search, execute, deploy
  • — Cross-project, full codebase awareness
"The model didn't change. The harness around it did."

03

Context is All You Need

CLI Agent 的一切工程化手段，都围绕一个核心：如何管理 context。

Input

Feed sufficient context

Brainstorming skills, paper-management agents, domain knowledge loading

Skills, MCP, CLAUDE.md

Output

Drive efficient decisions

Code review, automated pipelines, quality gates via hooks

Hooks, Subagents, Loop

Management

Keep the agent clear-headed

Internalize knowledge as docs, compress, delegate heavy work

Memory, /compact, Subagent

Skills, Hooks, Sub-Agent, MCP — all serve one purpose: engineering the context.
